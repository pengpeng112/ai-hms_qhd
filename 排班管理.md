## 二、数据结构

### 2.1 排班数据结构

```typescript
interface PatientShift {
  Id: number;           // 排班ID
  WardId: number;       // 病区ID
  BedId: number;        // 床位ID
  PatientId: number;    // 患者ID
  PatientPlanId: number; // 患者方案ID
  ShiftId: number;      // 班次ID
  ShiftTiming: number;  // 排班时效（10/20）
  Status: number;       // 状态（10/20/30/40/50/60）
  TreatmentTime: string; // 治疗日期
  LastModifyTime: string; // 最后修改时间
}
```

### 2.2 保存数据结构

```typescript
subSave = tdInfo
```

## 四、排班模板管理操作流程

### 4.1 模板管理界面功能

#### 4.1.1 模板切换

| 功能 | 说明 |
| :--- | :--- |
| 模板列表 | 系统支持多个模板（如正式模板、草稿模板） |
| 模板启用 | 可配置默认使用的模板 |
| 周数配置 | 支持2周/4周循环模板 |

#### 4.1.2 模板管理操作流程

```
选择模板 → 查看模板周数据 → 编辑模板 → 保存模板
    ↓
1. 从系统配置获取模板列表
2. 选择目标模板名称
3. 加载模板对应的周排班数据
4. 拖放调整患者排班
5. 保存到数据库
```

**查询对象格式**:

## 五、业务规则校验

### 5.1 时间规则

| 规则 | 校验逻辑 |
| :--- | :--- |
| 历史数据保护 | 治疗日期 < 当前日期 → 禁止修改 |
| 已过班次保护 | 当天班次排序 < 当前班次排序 → 禁止修改 |
| 治疗中保护 | Treatment_actions[20]存在 → 禁止删除 |

### 5.2 设备匹配规则

| 设备类型 | 支持治疗方式 |
| :--- | :--- |
| 血滤机 | HD, HDF, HF |
| 血透机（高通量） | HD, HFD |
| 血透机（低通量） | HD |


### 5.3 传染病患者规则

| 规则 | 校验逻辑 |
| :--- | :--- |
| 传染病患者 | 只能排到对应传染病区 |
| 普通患者 | 默认不能排到传染病区（可配置） |
| 多传染病 | 按优先级匹配病区 |

### 5.4 唯一性规则

- 同一时间同一床位只能有一位患者
- 同一患者在同一时间只能有一个排班记录
