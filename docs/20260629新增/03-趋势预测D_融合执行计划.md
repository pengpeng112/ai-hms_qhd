# 趋势预测 D 融合执行计划

## 1. 通俗功能说明

当前实时监控趋势线有一个简单外推：看最近几个点，大概画一条虚线到预计下机时间。这个项目把它升级为近窗最小二乘线性回归，让虚线预测更平滑、更不容易乱跳，并且让详情页趋势也显示预测虚线。

简单说：床卡和详情页的趋势虚线更稳定，但它不是低血压 AI，只是视觉趋势预测；真正事件预警仍由 IDH 模型负责。

## 2. 当前系统状态

- `extrapolateVitals` 仍是简单端点斜率外推。
- `GetTreatmentTrend` 当前只返回 actual 点，没有追加 predicted 段。
- 前端已有 `Kind=predicted` 的渲染基础，不需要改契约。

## 3. 融合目标

- 新增共享纯函数 `forecastSeries`。
- 床卡生命体征预测改用近窗最小二乘。
- 治疗详情趋势接口为各 series 追加 predicted 点。
- 保持 API 字段不变。

## 5. 执行步骤

### 5.1 新增预测纯函数

新增 `monitoring_forecast.go`，包含：

- `forecastPoint { T time.Time; V float64 }`
- `forecastOpts`
- `fitRecentLinear`
- `linearFit.eval`
- `forecastSeries`

算法要求：

1. 少于 2 点返回 nil。
2. `plannedEnd` 不晚于最后实际点返回 nil。
3. 默认取最近 6 点、最近 90 分钟内窗口。
4. 用最小二乘拟合 `V=a+b*t`。
5. 使用数据驱动钳位：窗口最小值/最大值外扩 0.5 倍跨度。
6. vitals 可传临床上下限，设备参数不传临床上下限。
7. 第一个 predicted 点为最后 actual 点，用作桥接。

### 5.2 替换床卡 `extrapolateVitals`

在 `monitoring_service.go` 中：

1. 保持函数签名不变。
2. 对 SBP、DBP、HR 分别取有效值序列。
3. 分别调用 `fitRecentLinear` 或 `forecastSeries`。
4. 临床钳位：SBP 50-220、DBP 30-130、HR 30-180。
5. MAP 由预测后的 SBP/DBP 派生。
6. 保持输出 `Kind:"predicted"`。

### 5.3 增强详情趋势 `GetTreatmentTrend`

在所有 actual series 构建完成后、`return out` 前追加：

1. **遍历 `out.Series` 的全部 key**（不要硬编码 key 列表），对每个 series 做预测：
2. 将 `[]TrendPoint` 转为 `[]forecastPoint`。
3. 临床钳位 map（命中则叠加临床上下限，不命中则仅数据驱动钳位）：
   - `sbp` 50-220、`dbp` 30-130、`map` 50-180、`heartRate` 30-180
   - 其余 key（`vp`、`ap`、`tmp`、`bf`、`conductivity`、`ufVolume` 及未来新增）不设临床上下限
4. 追加 `Kind:"predicted"` 的投影点，跳过桥接点，避免重复最后 actual。
5. 点数不足的 series 跳过（不产 predicted），不报错。

这种通用遍历策略确保未来新增 series key（如 `conductivity`、`ap`）自动获得 predicted 段，无需再改趋势代码。

### 5.4 新增测试

新增 `monitoring_forecast_test.go`，覆盖：

- 上升序列预测继续上升且被 envelope 钳位。
- 平稳序列预测近似平稳。
- 临床上限钳位生效。
- 点数不足返回 nil。
- plannedEnd 早于末点返回 nil。

## 7. 手工验收

- 实时监控床卡：已有多次体征的床位应出现 predicted 虚线点。
- 趋势详情：`GET /api/v1/monitoring/treatments/:id/trend` 的 `series` 中应同时有 `actual` 和 `predicted`。
- 点数不足或治疗已到 plannedEnd 后，不强行生成虚线。

## 8. 风险与不确定项

- 这是线性短时外推，不是医学事件预测；文案和代码注释都不能把它说成 IDH 预测。
- 设备参数采样密集，详情页已有 `trendMaxDevicePoints` 抽样；预测应基于抽样后的序列还是原始完整序列，当前交付包基于已进入 `out.Series` 的抽样序列。若要更准，需要另行设计。
- 若前端某些图表还没有按 `predicted` 渲染虚线，需要再查前端趋势组件；本计划默认前端已有支持。
