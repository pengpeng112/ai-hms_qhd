## 默认显示卡片（按位置 1-4）

| 位置 | 卡片 key | 标题 | 数据源（restApi 方法） | 备注 |
|---|---|---|---|---|
| 1 | todayShiftMatrix | 今日班次床位矩阵 | `getScheduleWeek({ date: today })` + `getBeds()` | 与护士共用 |
| 2 | pendingScheduleQueue | 待排班患者 | `getPatients({ noSchedule: true })` | 本周未排班患者 |
| 3 | bedUtilization | 床位利用率 | `getBeds()` + `getScheduleWeek()` | 按病区统计 |
| 4 | contractAnomaly | 排班合约异常 | `getClinicalTasks({ type: 'SCHEDULE_ANOMALY' })` | 需后端支持 |

## 取舍说明

- **为什么放这 4 张**：排班护士核心关注"今天排满了没"→"谁还没排"→"哪个区最紧张"→"有没有排错"
- **为什么不放设备/化验**：排班护士不直接管理设备，化验由医生关注
- **排班合约异常**：需后端支持 `SCHEDULE_ANOMALY` 任务类型，若无则降级为占位组件
