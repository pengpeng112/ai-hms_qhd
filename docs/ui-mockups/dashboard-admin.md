# Dashboard · ADMIN mockup

## 默认显示卡片（按位置 1-4）

| 位置 | 卡片 key | 标题 | 数据源（restApi 方法） | 备注 |
|---|---|---|---|---|
| 1 | operationKpis | 运营总览 4 KPI | `getStatistics({ type: 'overview' })` | 总患者/活跃/今日治疗/床位率 |
| 2 | deviceUtilization | 设备利用率 | `getDevices()` + 前端计算 | 在线率/使用率/报警率 |
| 3 | patientGrowth | 患者增长 | `getStatistics({ type: 'patient_growth', range: '30d' })` | 近 30 日趋势 |
| 4 | treatmentTrend | 治疗量趋势 | `getStatistics({ type: 'treatment_trend', range: '30d' })` | 近 30 日趋势 |

## 取舍说明

- **为什么放这 4 张**：管理员关注全局运营效率，KPI 总览 + 设备 + 患者 + 治疗形成管理闭环
- **为什么不放排班/医嘱**：这些是执行层关注，管理员通过统计报表查看趋势即可
- **患者增长/治疗量趋势**：若统计接口不支持 `type` 参数，降级为全局统计概览
