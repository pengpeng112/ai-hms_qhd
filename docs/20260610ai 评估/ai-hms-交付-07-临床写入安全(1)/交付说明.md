## 三、设计要点（详见 docs/临床写入安全加固设计.md）

- **乐观锁**两档：整型 `Version` 列（最稳）/ `LastModifyTime` 时间戳（零改表过渡）；冲突→HTTP 409。
- **前端契约不变**：版本令牌走 HTTP `ETag` / `If-Match` 头，JSON 响应体零改动，前端改造收敛于
  `restClient.ts` 单点；`If-Match` 缺失时过渡期放行，灰度后切严格。
- **幂等**：自然键唯一约束优先（复用排班 `scripts/schedule_unique_indexes.sql` 套路），
  无自然键则用独立 `App_IdempotencyKey` 表。
- **审计**：应用层独立 `Audit_ClinicalWrite` 表即可起步；老系统并行写入的全覆盖需评估 DB 触发器。
- **铁律**：所有建列/建表/建索引走外部 SQL + DBA + 重复数据探测，代码零 DDL；落地每张表前必对 schema 文档。
