# 透析排班子程序 规则规范 v1

> AI 透析管理系统子程序:为每位透析病人安排未来 2~4 周的透析治疗(区域 + 机器 + 班次)。
> 本规范由刘子栋医生与 AI 逐条讨论确定(共 20 条决策),作为开发依据。
> 配套:`新规则决策记录.md`(讨论过程)、`透析排班子程序规则.docx`(原始规则)、`排班规则_老对新对比.md`(老系统对照)。

---

## 0. 两条贯穿全局的铁律

1. **主方案 → 备选方案 → 报警人工裁决**
   排班算法不追求"自动解决一切冲突"。任何约束无法自动满足时,宁可报警、放入冲突队列交人工决定,也不强行乱排。

2. **算法只读"时间骨架",只负责"分机位"**
   病人的分区、频率、班次、治疗模式都是**人工权限登记属性**,算法不决定、只读取。算法的唯一职责:在骨架(区+天+班+模式)约束下,为每一次透析分配具体的机器/床位。

## 1. 资源模型

### 1.1 分区

- 透析室分 **A、B、C 区,可扩展**;每区可再分子区 A1/A2/A3…、B1/B2/B3…。
- **A 区 = 门诊病人区;B 区 = 住院病人区;C 区 = 全警戒医疗区。**
- A 区病人增多时拆分子区。

### 1.2 机器(机型与能力)

- 每台机有唯一标识,机型分 **HD / HDF / CRRT**。
- 能力包含关系:
  - **HD 机**:只能做 HD。
  - **HDF 机**:能做 HD、HDF、HF(向下兼容)。
  - **CRRT 机**:只能做 CRRT。
- 机器另有属性:所属区、停用时段(见 §9.5)。

### 1.3 班次与位

- 每周 **周一至周六**(周日默认非透析日);每天三班:**上午班 / 下午班 / 晚班**。
- 班次约束:上午班只排上午、下午班只排下午、晚班只排晚班;临时透析不受此约束。
- **容量模型:一个班次 × 一台机 = 1 个透析位,班次内不翻台。**一台机一天透三人由三个班次表达。
- 区容量 = 区内机器数 × 班次数。**无"加床"概念**(排满时顺延到后面时段找空位,见 §9.4)。

## 2. 病人时间骨架(人工权限属性,算法只读)

| 维度 | 取值 | 谁决定 | 权限 |
|------|------|--------|------|
| 分区 | A / B / C(标签) | 医生 | 医生/护士长改标签 |
| 频率模式 | 见 §2.1 五种 | 医嘱 | 医嘱 |
| 班次 | 上午 / 下午 / 晚(全周同一班) | 登记指定 | 护士长、主班护士 |
| 治疗模式 | HD / HDF / CRRT(按次) | 医嘱 | 医嘱 |
| HDF 日 | 病人某个透析日 | 见 §5 | 医嘱/系统测算 |

### 2.1 五种频率模式

| 模式 | 频率 | 固定排在 |
|------|------|----------|
| ① 一三五 | 每周 3 次 | 周一、三、五 |
| ② 二四六 | 每周 3 次 | 周二、四、六 |
| ③ 每周两次 | 每周 2 次 | 周二、周四 |
| ④ 每周一次 | 每周 1 次 | 周四 |
| ⑤ 临时 | 不定 | 任意时间(见 §9.1) |

### 2.2 班次规则

- 班次是病人**固定登记属性**,**所有透析日同一个班**。
- 登记时人工指定,系统只读取;系统不自动给病人挑班次。
- 指定/修改班次的权限:**仅护士长、主班护士**。

## 3. 分区与院感规则

- **分区由病人标签驱动,系统只读不改**:门诊→A、住院→B、C 区病人在输入时即带 C 区标签。
- **C 区(全警戒区)收治**:危重病人、新病人、无明确院感指标者、急症透析、传染病急诊人道救治。
  - "无明确院感指标"= 指标查清前先进 C 区(预防性),查清阴性后转出。
- **传染病**:本院**不接收明确传染病病人**(确诊一律拒收);例外:出于人道主义的**急诊救治**可安排在 C 区。**不设独立传染病区,不做按病种隔离**。
- **拒收**:门诊+传染病 → 拒绝收治(不生成排班);可由医生/护士长权限将其标识为 C 区特征病人,改走 C 区。
- **转区**:C 区转 A/B,**必须由医生先明确变更病人分区标签**,标签改后系统方可据此排班。系统不自动转区(可提示"院感结果已阴性,建议考虑转出")。
- **跨区**:默认**严格不跨区**;跨区只能作为护士长权限的特批例外(见 §9.4)。

## 4. 机位分配算法

### 4.1 两轮分配(核心骨架)

> 在每个"区 × 班次 × 日期"的格子里分配机器时,按两轮进行:

- **第一轮(HDF 次)**:把所有"该做 HDF 的次"先排进 HDF 机(按 §5)。
- **第二轮(HD 次)**:HD 次**优先排 HD 机**;**HD 机全满后**,才把 HD 次溢出到空闲的 HDF 机位。

→ 保证 HDF 机既为 HDF 病人留有保障,又不闲置浪费。

### 4.2 固定机位(老病人)

- 每个病人尽量排在其**上一次使用的同一台机器**(保持固定位置);否则选**最靠近**的机器。
- 模板复制时,稳定病人原样沿用固定机位。

### 4.3 新病人初始分配

1. **首选**:找一台该病人**所有透析日都空闲的同一台机**,一次性固定。
2. **退路**:找不到三天全空的同一台机 → **逐次分配**(各日分别找空位,可能不同机)。
3. **报警**:仍排不开 → 报警人工。

### 4.4 挑机偏好(多台候选时)

- **集中**:优先让空机**连成片**,保护"整台全空机"这种稀缺资源。
- **组团**:同班次、同方案病人尽量排在**相邻机位**,便于护士分组管理。
- **不做负载均衡**(刻意打散会破坏位置连续性)。

## 5. HDF 专项规则

- **替换语义**:"每两周一次 HDF" = 把病人某一次 HD **替换为 HDF**,总透析次数不变。治疗模式是**每一次排班记录**的属性。
- **HDF 日**:固定在病人某个透析日(必须是其已有透析日);做 HDF 的是奇周还是偶周由**系统按 HDF 机位跨周均匀性测算**决定(错峰),一旦排定尽量固定,机位实在不够才微调时间。
- **奇偶周定义**:设全局**固定基准周一**(配置项,默认 2025-01-06);任意周序号 =(该周周一−基准周一)÷7,奇偶 = 序号 % 2。连续计数、跨年无缝、永不漂移。每个 HDF 病人有 `HdfWeekParity ∈ {奇,偶}`,在人少的一侧入组。(奇偶周仅服务 HDF 错峰;5 种频率模式本身不用单双周。)
- **HDF 机位(优先级链)**:
  1. **双固定(首选)**:病人平时固定一台 HD 机,做 HDF 时固定回到**同一台 HDF 机**。
  2. **就近(退路)**:固定 HDF 机被占 → 就近调度一台空闲 HDF 机。
  3. **报警**:无空闲 HDF 机 → 报警人工。
- **HDF 优先**:排位时 HDF 模式是最高优先考量;HDF 病人始终先于 HD 病人占用 HDF 机(见 §4.1 第一轮)。

## 6. 排班生成

- **生成方式 = 模板复制**:维护标准模板,生成时把稳定病人的模式原样往后铺;只有变动病人(新入/转出/改方案/请假)单独处理。不做全量重排。
- **周期**:默认**每 2 周**生成;界面提供**"4 周"选择按钮**(4 周 = 2 个 HDF 周期)。
- **模板独立存储**:模板单独存表/单独标记,**不**用状态值兼任(根除老系统"删排班误删模板"的坑)。

### 6.1 三级确认工作流

| 确认 | 时机 | 谁 | 对象 |
|------|------|-----|------|
| 第一次 | 草稿生成后 | 护士长 | 整个 2/4 周计划 → 生效 |
| 第二次 | 透析日**前一天 11:00 前**(周一的在**前一个周六**) | 主班/护士长 | 次日全天排班 |
| 第三次 | 当日**早上** | 主班/护士长 | 当日全天排班 |

- 三级确认之间允许按现实微调(请假、急诊加台等),每级是一道纠错关口。
- 第二次确认需处理**非工作日跳转**(周日不排,周一的次日确认提前到周六)。

## 7. 状态机与数据形态

### 7.1 排班记录状态机

| 编码 | 状态 | 含义 |
|------|------|------|
| 0 | 待排 | 应排未排(缺口) |
| 10 | 草稿 | 模板复制/算法生成,未确认 |
| 20 | 已确认 | 计划生效;附 3 个时间戳 `confirm1_at`/`confirm2_at`/`confirm3_at` |
| 50 | 透析中 | 已上机 |
| 60 | 已完成 | 本次结束 |
| 70 | 已取消 | 提前请假/计划取消(留痕) |
| 80 | 缺席 | 当日爽约(留痕,机位可借) |

### 7.2 正交字段(不进状态机)

- `sourceType`:常规(模板复制)/ 临时(急诊插入)
- `dialysisMode`:HD / HDF / CRRT(按次)
- **"拒收"是病人登记环节的病人级结果**,不生成排班记录。

### 7.3 数据形态

- 排班记录支持两种形态:
  - **①规律透析**:区 + 机器/床 + 班次 + 日期,走状态机 / 模板。
  - **②CRRT**:机器 + 治疗起止时间区间,特殊类型(见 §10)。

## 8. 历史与编辑保护(继承老系统)

- 过去日期的排班、已开始治疗(透析中/已完成)的排班**不可修改/拖拽**。
- 冲突检测:同一病人在同一 `日期 + 班次` 不可重复排班。
- 差异检测(常驻):提示每病人"本周应排 X 次、已排 Y 次、差 Z 次"。

---

## 9. 扰动处理(均走"系统建议 + 人工确认",不自动定)

### 9.1 临时透析(第五种)

- 当前班次、对应区就近找空闲机器插入(C 区急症/危重优先用 C 区空位),按医嘱模式匹配机型。
- 当前班次无空位 → 看下一班次 → 仍无 → 报警。
- 可借用当天请假病人空出的机位(当天一次性,不改原归属);不得挤走在排的规律病人。
- **不进模板**;权限操作(急诊医嘱 + 护士长/主班确认)。

### 9.2 请假 / 缺席

- **提前请假**:该次标记取消;是否补、补哪天哪班由医生/护士长人工决定(系统给空位建议,不自动补)。
- **当天缺席**:标记缺席(留痕),机位释放可被借用。
- **系统不自动补透**,只提示次数差异。

### 9.3 方案(医嘱)变更

- 带**生效日**(默认次日或下一个未确认排班日,人工确认)。
- 生效日后、**尚未二/三次确认**的排班 → 按新方案**重排为草稿**,重走确认。
- 已临近、**已二/三次确认**的近期排班 → 不自动改,需改走人工。
- 改模式触发 HDF 机位逻辑(§5);改分区须医生改标签(§3);尽量保留原 HD 机位。
- 高亮提示"该病人 X 日起方案变更,N 条未来排班受影响,请审核"。

### 9.4 排满 / 跨区

- **无"加床"概念**。排满时的处理 = **在后面的时段(后续班次 / 后续日期)里选有空位的机器安排**(时间维度顺延,不在空间上加位)。
- 排不进当前时段 → 系统建议**顺延到后面可用时段** → 护士长权限确认;协调改班次/改日期或**特批跨区**也是选项。
- 完全无可用时段 → 报警,放入**待处理/冲突队列**。
- 跨区、改班/改日期均为人工权限操作,系统不自动执行。
- 系统**实时显示每区每班余位**,余位 < 阈值时预警。

### 9.5 设备故障 / 维修

- 工程师/护士长权限登记机器"停用"及**停用时段**,系统冻结该机该时段的位。
- **临时停机(默认 ≤ 48 小时)**:系统自动列出受影响排班,为每病人就近找替代空位生成调整草稿 → 护士长确认;**不改原固定机位,修好后自动归位**。
- **长期停用 / 报废(>48 小时)**:受影响病人由人工重新分配固定机位(永久迁移)。

### 9.6 法定假日

- 维护**"非透析日"日历**(权限),生成排班自动跳过(周日默认非透析日)。
- 受影响病人:系统给"就近前后挪"建议草稿 → 人工确认。
- 支持**假日值班模式**:假日可只开放部分区/机器做急诊透析。

## 10. CRRT 特例

- CRRT 纳入本系统,作为 **C 区的特殊排班类型**。
- **不走三班网格、不走模板、不走固定机位**;只记录占用哪台 CRRT 机 + 治疗起止时间(可跨班、跨天)。
- CRRT 机只做 CRRT;CRRT 病人属 C 区(危重);权限操作。
- CRRT 占用须在视图显示,避免该机被误排。

## 11. 权限矩阵(汇总)

| 操作 | 权限角色 |
|------|----------|
| 改病人分区标签(含 C↔A/B、拒收转 C) | 医生 / 护士长 |
| 指定/修改病人班次 | 护士长 / 主班护士 |
| 频率 / 治疗模式 / HDF(医嘱) | 医生(医嘱) |
| 第一次确认(计划生效) | 护士长 |
| 第二、三次确认 | 主班护士 / 护士长 |
| 加床 / 跨区特批 / 改班协调 | 护士长 |
| 登记设备停用 | 工程师 / 护士长 |
| 维护非透析日日历 | (机构管理权限) |
| 排入临时透析 / CRRT | 医生(医嘱) + 护士长/主班确认 |
