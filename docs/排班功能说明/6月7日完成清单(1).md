## 二、P0:数据库唯一约束 + 并发防护

**目的**:此前"同病人同日同班不重复""同机同日同班一人"只靠应用逻辑,并发可击穿。补数据库级安全网。

**改动**:

- `internal/db/db.go`:迁移时创建两个 PostgreSQL **分区唯一索引**
  - `uq_ps_machine_slot`(同机+同日+同班一人)
  - `uq_ps_patient_slot`(同病人+同日+同班不重复)
  - 条件 `WHERE Status NOT IN (70,80) AND ... NOT NULL`(取消/缺席不计,CRRT 排除)
- `internal/repo/repo.go`:批量写草稿用 `ON CONFLICT DO NOTHING`(并发幂等)。
- `internal/service/ops_service.go`:`isUniqueViolation`(识别 23505)→ 移床转"目标位被占用"、临时透析换班次重试、补透跳过。

**验证(全新库)**:索引建立成功;直接 INSERT 重复行被数据库拒(两条约束都生效);401/403/200 鉴权正常;`/health` 正常。

## 三、P1:录入维护 + 模板 + 冲突处理

**目的**:此前只能靠演示数据(seed),无法录入真实科室资源与病人。补全"从演示到可用"。

**改动**:

- `internal/service/admin_service.go`:病区 / 机器 / 班次 / 病人主档 / 排班骨架 的 增改查停,含校验
  (分区 A/B/C、机型 HD/HDF/CRRT、HDF 日须落在频率透析日内等)。
- `internal/service/template_service.go`:`RebuildTemplateFromProfiles`(把当前病人骨架快照为新的生效模板)、
  模板/模板项查询、`ResolveConflict`(accept=已处理 / ignore=已忽略)。
- `internal/api/api_admin.go`:全部挂 `/admin/*`(读不设守卫,写按角色守卫);`/conflicts/:id/resolve`。
- 前端 `web/index.html`:新增「⚙ 管理」面板(病区 / 机器 / 病人+骨架 / 模板 四标签),网页内即可维护。

**验证**:在全新空库里**完全用接口**搭出一个透析室(病区→班次→机器→病人+骨架→重建模板→生成排班→处理冲突);
非法骨架被拒;Playwright 渲染四个标签均正常。
