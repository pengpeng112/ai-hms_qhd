# 排班管理与排班模板管理规则逻辑文档

## 一、概述

本系统是血液净化中心智能透析管理系统的排班管理模块，包含两个核心功能：
- **排班管理**：实际患者排班的日常管理
- **排班模板管理**：排班模板的配置与维护

两者通过模板复制机制关联，模板提供周期性排班的基准配置。

## 二、排班管理规则

### 2.1 界面布局结构

排班管理界面采用**矩阵式日历视图**：
- **行维度**：病区 → 床位（按病区分组显示）
- **列维度**：日期 → 班次（横向展示一周数据）

### 2.2 时间计算规则

```typescript
// 获取指定周的日期范围
```

## 三、排班模板管理规则

### 3.1 模板周期配置

系统支持**多周循环模板**，默认配置为4周：

```typescript
patientShiftWeekTPL = 4; // 排班模板周数
```

模板通过**基准年份**区分：
- **正式模版**：基准年份 2000 年
- **草稿模版**：基准年份 2010 年

### 3.2 模板时间计算规则

## 四、设备匹配规则

### 4.1 设备类型分类

| 设备类型 | 支持的治疗方式 | 特殊属性 |
|----------|---------------|----------|
| 血透机 | HD | 高通量/低通量 |
| 血滤机 | HD、HDF、HF | 无 |

### 4.2 治疗方式匹配逻辑

```typescript
// 过滤只能做血滤机的治疗方式
xlfilterDialysisMethod(dialysisMethod) {
  let type = ['HDF', 'HF'];
  return type.some(method => dialysisMethod.includes(method));
}

// 血透机高通量检测
xtDeviceHFDDialysisMethod(dialysisMethod) {
  return dialysisMethod.includes('HFD');
}
```

### 4.3 床位分配优先级

```
1. 优先选择同方案同周几的已有床位
2. 其次选择同病区的空床
3. 最后选择其他病区的空床
```

### 5.1 历史数据保护


### 5.2 设备兼容性校验

```typescript
examBedEquipmentAndPlanDialysisMethod(value, bedId, ...) {
  let deviceType = bedList[0].Type;
  
  // 血透机不能做 HDF/HF
  if (deviceType != '血滤机' && xlfilterDialysisMethod(value.DialysisMethod)) {
    return { isSave: true, mesage: '设备不支持该治疗方式' };
  }
  
  // 低通量血透机不能做 HFD
  if (deviceType == '血透机' && xtDeviceHFDDialysisMethod(value.DialysisMethod)) {
    // 检查是否为高通量
    if (!bedList[0].Flux === '高') {
      return { isSave: true, mesage: '低通量设备不支持 HFD' };
    }
  }
}
```

## 八、总结

排班管理系统通过以下核心机制实现智能排班：

1. **周期模板机制**：支持多周循环模板，减少重复排班操作
2. **差异检测**：自动识别应排与已排的差异，提醒遗漏排班
3. **设备匹配**：根据治疗方式智能分配兼容设备
4. **病区隔离**：基于传染病检查结果自动分配病区
5. **历史保护**：禁止修改已完成或进行中的排班数据
6. **双模板机制**：正式模板用于实际排班，草稿模板用于测试和预览
