# 透析排班规则：老规则 → 新规则 对比

> 新 AI 透析排班子程序：为每位病人安排未来 2~4 周的透析（区域 + 机器 + 班次）。
> 本文先梳理**老程序已实现的规则**，再梳理**你新写的规则**（`透析排班子程序规则.docx`），最后做差异与待决清单。

# 第一部分：老程序已实现的规则

来源：`老系统排班业务逻辑.md`、`schedule-module-guide.md`、`schedule_source.txt`（Go 后端 + React 前端）。
老程序偏重**实现机制**，对“怎么决定病人去哪台机器”的业务策略描述较少。

## 1. 数据与界面
- 矩阵式日历视图：行 = 病区→床位，列 = 日期→班次，横向展示一周。
- 核心表 `Schedule_PatientShift`：一行 = 患者×病区×床位×班次×日期 + 状态。
- 资源表：`Schedule_Ward`（病区）、`Schedule_Bed`（床位，含 Type 血透/血滤、Flux 高/低通量）、`Schedule_Shift`（班次）、`Schedule_BedEquipmentRel`（床位-设备绑定）。

## 2. 时间与频率
- 按周加载；`周数 % 2` 判断单/双周。
- 应排次数按单/双周取 `OddWeekFrequency` / `EvenWeekFrequency`。
- 差异检测：`应排次数 != 已排次数` 标记遗漏。

## 3. 模板机制
- 多周循环模板，默认 4 周（`patientShiftWeekTPL = 4`）。
- 双模板：正式模版基准年 **2000**，草稿模版基准年 **2010**。
- 实际排班为空时提示从模板复制；目标周对应模板周 = `周数 % 4`。
- 模板复用主表，靠 `Status = 60` 区分（⚠️ 删除排班易误删模板）。
- 转出患者：批量软删除其在模板各周排班。

## 4. 设备兼容校验（已写成代码）
- 血透机不能做 HDF/HF；低通量血透机不能做 HFD（需高通量）。
- 病区隔离：依传染病 LIS 结果分配传染病/普通病区。

## 5. 床位分配优先级（老代码注释）
1. 同方案、同周几的已有床位；2. 同病区空床；3. 其他病区空床。

## 6. 状态机与编辑保护
- 状态：草稿(10)→已确认(20)→签到/透析(30)→治疗中/完成(40)；取消(50)/删除(60)。
  （⚠️ 30/40/50 中文标签在不同文件不一致，需统一。）
- 历史保护：过去日期、已开始治疗的排班不可改/拖拽。
- 冲突检测：同 `日期+班次` 不可重复排同一病人。
- 互换在事务内交换两条记录字段；换床改 `BedId`。

# 第二部分：你新写的规则（docx）

来源：`透析排班子程序规则.docx`。这是**业务策略层**的规则，补齐了老程序缺的“排位决策逻辑”。

## 1. 分区规则
- 透析室分 A、B、C 区，可扩展；子区 A1/A2/A3…、B1/B2/B3…。
- **A 区 = 门诊病人，B 区 = 住院病人，C 区 = 全警戒医疗区**。
- A 区病人增多时拆 A1/A2/A3。
- **C 区只收四类病人**：危重、新病人、无明确院感指标、急症透析。
- **一般不跨区**安排病人。

## 2. 机器类型与能力
- 每台机有唯一标识，类型 HD / HDF / CRRT。
- **HD 机只做 HD；HDF 机能做 HD、HDF、HF；CRRT 机只做 CRRT。**

## 3. 时间与班次
- 每周**周一到周六**，每天三班：**上午班 / 下午班 / 晚班**。
- 上午班只排上午、下午班只排下午、晚班只排晚班；**临时透析可随时安排**。

## 4. 五种频率模式
| 模式 | 频率 | 固定排在 |
|------|------|----------|
| ① | 每周 3 次 | 周一、三、五 |
| ② | 每周 3 次 | 周二、四、六 |
| ③ | 每周 2 次 | 周二、周四 |
| ④ | 每周 1 次 | 周四 |
| ⑤ 临时 | 不定 | 任意时间 |

## 5. 排班周期
- 一般 **每 2 周 或 每 4 周**给病人排一次，**通常是重复排班**。

## 6. 治疗模式与机器匹配（新规则核心）
- 多数病人为 **HD 模式**，可上 HD 机，也可上 HDF 机。
- **每个病人每两周安排一次 HDF**，且该次只能上 HDF 机。
- **HDF 优先级**：HD/HDF 机选择时，HDF 机优先满足 HDF 模式病人 → 排位时 HDF 模式是最高优先考量。

## 7. 位置稳定性
- 每个病人**尽量排在上一次用的那台机器**（固定位置），否则选**最靠近**的机器。

# 第三部分：新规则相对老程序的“新增/变化”

| 维度 | 老程序 | 新规则（你写的） |
|------|--------|------------------|
| 分区语义 | 仅有病区表 + 院感隔离 | 明确 A=门诊/B=住院/C=全警戒，C 区四类病人，不跨区 |
| 机器能力 | 血透/血滤 + 通量校验 | 明确 HD/HDF/CRRT 三类及能力包含关系 |
| 班次 | 班次表（数据驱动） | 固定三班（上午/下午/晚）+ 周一到周六 |
| 频率 | 单/双周 frequency 字段 | 明确五种固定模式（含临时） |
| HDF 策略 | 无 | **每 2 周 1 次 HDF + HDF 机优先级**（全新核心策略） |
| 位置策略 | 床位优先级 3 档 | **固定位置/最近机器优先**（病人黏性） |

# 第四部分：待你确认 / 新规则尚需明确的点

1. **优先级仲裁**：当“固定位置”与“HDF 优先”冲突（病人上次在 HD 机、本次轮到 HDF），谁让步？需要一条明确的权重/排序。
2. **HDF 周期对齐**：在 2 周 / 4 周模板里，“每两周一次 HDF”落在第几周第几次？需与频率模式（一三五等）对齐。
3. **临时透析落位**：第五种“可随时”——落哪台机、是否可占固定病人位、是否计入容量？
4. **C 区流转**：危重/新病人稳定后何时、如何迁回 A/B 区？
5. **容量溢出**：某区某班机器排满时的策略（候补 / 跨区例外 / 报警）？
6. **跨区例外**：“一般不跨区”——哪些情形允许例外（如 C 区满）？
7. **状态枚举统一**：老系统 30/40/50 标签不一致，新系统须先定义唯一口径。
8. **方案/干体重变更**对已排未来排班的回溯影响。
