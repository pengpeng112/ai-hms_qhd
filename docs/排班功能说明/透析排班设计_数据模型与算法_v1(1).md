# 透析排班子程序 — 数据模型与算法设计 v1

> 配套规范:`透析排班子程序规则规范_v1.md`(权威,22 条决策结晶)。
> 本文是规范的**技术落地设计**:数据模型(表结构)+ 排班算法伪代码 + 设计取舍 + 开放问题。
> 原则:**忠于规范 v1,不改业务规则**。发现规范矛盾/缺口的,统一记入 D 部分,不擅自决定。
> 技术栈对齐老系统(ai-hms):Go + Gin + GORM + PostgreSQL,表在 `Schedule_` 命名空间;多租户字段 `TenantId` 贯穿全表。

## 0. 命名与通用约定

- 所有表带 `TenantId bigint`(多租户,沿用老系统),所有查询/唯一约束默认带 `TenantId` 前缀,下文不再逐一重复。
- 主键统一 `Id bigint`(雪花/序列),`CreatorId / CreateTime / LastModifyTime` 审计三件套沿用老系统。
- 软删除统一用 `IsDisabled boolean`(资源类)或状态机(排班记录),**不物理删**。
- 时间:日期用 `date`,时间戳用 `timestamptz`。班次用班次表外键,不用裸字符串。
- 新设计与老系统最大差异(规范强制):
  1. **模板独立存表**(`Schedule_ScheduleTemplate`),**绝不**再用 `Status=60` 兼任模板(根除"删排班误删模板")。
  2. 取消老系统的"传染病病区/院感隔离子区"逻辑(决策 6),分区简化为 A/B/C + 子区。
  3. 治疗模式 `DialysisMode` **按次**存(决策 1),不再是病人固定属性。
  4. 病人新增"分区标签 / 频率模式 / 固定 HD 机 / 固定 HDF 机 / HDF 日"等排班骨架属性。

## 0.1 系统参数 / 配置项(租户级)

> 排班算法依赖的可配置全局参数。建议落在租户级配置表(沿用老系统配置机制)或 `Schedule_TenantSetting`,代码读取时带默认值兜底。

| 配置键 | 类型 | 默认值 | 说明 |
|------|------|------|------|
| `OddEvenWeekAnchorMonday` | date | **2025-01-06**(周一) | **奇偶周基准周一**(决策 21 / D-6)。任意周序号 =(该周周一 − 基准周一)÷ 7;奇偶 = 序号 % 2。连续计数、跨年无缝、永不漂移。仅服务 HDF 错峰;5 种频率模式本身不用单双周。详见算法 `week_index/week_parity`。 |
| `LowSlotWarnThreshold` | int | 待定(D-1) | 余位预警阈值:区/班余位 < 此值触发预警(§9.4)。 |
| `DraftWeeks` | int | 2 | 一次生成草稿的周数(2 或 4,§6 / 决策 11)。 |

- **不变量**:`OddEvenWeekAnchorMonday` 必须是周一(应用层校验 `weekday()==1`)。一旦上线**不应再改**(改它会让全体 HDF 病人奇偶周翻转);如确需改,须配套迁移所有 `HdfWeekParity`。

### A.3.1 排班记录 `Schedule_PatientShift`(对应老表,**重构状态机**)

  - 缺席(80)与取消(70)分离,机位可借由 `IsBorrowedSlot` 表达。

### A.3.2 CRRT 占用 `Schedule_CrrtSession`(**新表**,决策 18 / 规范 §10)

> CRRT 不走三班/模板/固定机位,只记机器+起止时间(可跨班跨天)。可作为 PatientShift 的扩展(RecordForm=20)挂一条,也可独立成表。本设计**独立成表 + PatientShift 主记录**(主记录走状态机,扩展记录存时间区间),便于统一视图与冲突检测。

| MachineId | bigint | NOT NULL, FK→Machine | 必为 CRRT 机型 |
| StartAt | timestamptz | NOT NULL | 治疗开始 |
| EndAt | timestamptz | NULL | 治疗结束(进行中为 NULL) |

```python
def generate_schedule(tenant, start_date, weeks=2):
    """主流程:模板复制生成未来 2 或 4 周草稿。weeks ∈ {2,4}。"""
    template = get_active_template(tenant)                 # 独立模板表,绝不读 Status=60
    dates    = expand_dialysis_dates(tenant, start_date, weeks)
    # ↑ 按机构日历跳过非透析日(周日默认非透析日,§9.6/决策19)

    # 1) 先为启用了 HDF 的稳定病人测算"HDF 落在奇周还是偶周"(决策3,跨周均匀)
    assign_hdf_week_parity(tenant, template, dates)

    # 2) 逐 (日期, 区, 班次) 网格生成草稿
    for d in dates:
        if is_holiday_duty_mode(tenant, d):
            handle_holiday(tenant, d)        # 假日值班模式:只开放部分区/机(§9.6),受影响病人挪班建议
            continue
        for ward in open_wards(tenant, d):                # 假日模式只取开放区
            for shift in shifts(tenant):
                cell = Cell(ward, shift, d)
                items = template_items_in(tenant, template, ward, shift, due_on=d)
                # ↑ 按 FreqPattern 推导:该病人该日是否应透 + 本次模式(HD/HDF)
                place_cell(tenant, cell, items)           # 两轮分配,见 B.1

    # 3) 识别缺口(应排未排)→ 写 Status=0 待排 + 差异检测
    detect_gaps_and_diffs(tenant, dates)
    # 4) 全部草稿 Status=10,等护士长第一次确认(B.4)
```

```python
def expand_dialysis_dates(tenant, start, weeks):
    out = []
    for i in range(weeks * 7):
        d = start + days(i)
        if is_dialysis_day(tenant, d):     # 查 Schedule_Calendar;无记录则周日=False 其余=True
            out.append(d)
    return out

def template_items_in(tenant, template, ward, shift, due_on):
    """返回该格子里、该日应透析的病人,并标注本次模式。"""
    result = []
    for it in template.items_of(ward, shift):
        if not is_due(it.freq_pattern, due_on):                 # 一三五/二四六/每周两次/每周一次 → 星期匹配
            continue
        mode = decide_mode_of_this_session(tenant, it, due_on)  # HDF 替换判定(用全局奇偶周锚点),见 B.2
        result.append(SessionItem(patient=it.patient, mode=mode, template_item=it))
    return result
```

## B.1 核心:两轮分配(规范 §4.1,决策 4)

```python
def place_cell(tenant, cell, items):
    """在一个 (区×班×日) 格子内分配机器。两轮:先 HDF 次进 HDF 机,再 HD 次。"""
    hdf_items = [x for x in items if x.mode == 'HDF']
    hd_items  = [x for x in items if x.mode == 'HD']

    # ── 第一轮:HDF 次,先占 HDF 机(为 HDF 病人留保障) ──
    for it in hdf_items:
        machine = place_hdf_session(tenant, cell, it)     # 双固定→就近→报警,见 B.3
        if machine is None:
            raise_conflict('HDF_NO_MACHINE', it, cell)    # 报警,不硬排
        else:
            draft(it.patient, cell, machine, mode='HDF', source='REGULAR', template_item=it.template_item)

    # ── 第二轮:HD 次,优先 HD 机,满了溢出到空闲 HDF 机 ──
    for it in hd_items:
        machine = place_hd_session(tenant, cell, it)
        if machine is None:
            # 本格(区×班×日)排满:不加床(决策22)。沿"时间维度顺延"到后续班次/后续日期找空位。
            spill = spill_to_later_slot(tenant, it, cell)        # 见 B.1.1
            if spill is None:
                raise_conflict('NO_MACHINE', it, cell)           # 完全无可用时段 → 报警入队人工裁决(§9.4)
            # spill 命中时已在内部生成"建议草稿"并写提示队列,等护士长人工确认
        else:
            draft(it.patient, cell, machine, mode='HD', source='REGULAR', template_item=it.template_item)
```

```python
def place_hd_session(tenant, cell, it):
    """HD 次落位:固定机 → 同区就近 → 溢出 HDF 机。全用挑机偏好打分(B.4)。"""
    p = it.patient
    # 1) 老病人:优先上一次同台 HD 机(固定位置,§4.2)
    if p.fixed_hd_machine and is_free(p.fixed_hd_machine, cell) and not_outage(p.fixed_hd_machine, cell):
        return p.fixed_hd_machine
    # 2) 同区空闲 HD 机,按"集中连片+组团"打分挑(§4.4)
    hd_free = free_machines(cell, type='HD')
    if hd_free:
        return pick_best(hd_free, p, cell)        # B.4 挑机偏好
    # 3) HD 机全满 → 溢出到空闲 HDF 机(决策4),同样按偏好挑
    hdf_free = free_machines(cell, type='HDF')
    if hdf_free:
        return pick_best(hdf_free, p, cell)
    return None                                   # 本格排不开 → 上层走"时间顺延"(B.1.1)再不行才报警
```

## B.1.1 排满处理:时间维度顺延找空位(决策 22 / 规范 §9.4)

> **系统中不存在"加床"概念**,不新增临时位/机动机,容量模型恒为"机器数 × 班次数"。
> 当前格(区×班×日)排满时,**在后面的时段里**(先同日后续班次,再后续日期同班)为该病人找有空位的机器,
> 走"系统建议 + 护士长人工确认";**完全无可用时段**才报警入冲突队列。

```python
def later_slots(tenant, cell, horizon_days=14):
    """生成"后续时段"序列:先同日后续班次(按 Sort 升序),再后续透析日(优先病人原班次,
       同日内仍按班次 Sort 兜底)。只在草稿窗口/合理时限(默认 14 天)内顺延,不无限外推。"""
    # 1) 同一天,排在当前班次之后的班次
    for sh in shifts_after(tenant, cell.shift):
        yield Cell(cell.ward, sh, cell.date)
    # 2) 后续透析日(跳过非透析日),优先回到病人原班次,再退到其它班次
    for d in dialysis_days_after(tenant, cell.date, horizon_days):
        yield Cell(cell.ward, cell.shift, d)                  # 先试原班次(尽量不改病人作息)
        for sh in other_shifts(tenant, cell.shift):
            yield Cell(cell.ward, sh, d)

def spill_to_later_slot(tenant, it, cell):
    """本格排满 → 按"后续班次 / 后续日期"顺序找该病人可用的空位机器;
       命中则生成"建议草稿"(待护士长确认),否则返回 None 交上层报警。"""
    for cand in later_slots(tenant, cell):
        if not patient_available(it.patient, cand):           # 病人在该时段无冲突(无已有有效排班)
            continue
        m = place_hd_session(tenant, cand, it)                # 复用落位逻辑(固定机→就近→溢出HDF机)
        if m is not None:
            s = draft(it.patient, cand, m, mode='HD', source='REGULAR',
                      template_item=it.template_item)         # 顺延位置生成草稿
            raise_conflict('SLOT_SPILLED', it.patient, cand,  # 提示级:本次因原时段满而顺延
                           severity=HINT, suggested=s,
                           detail={'origin': cell, 'spilled_to': cand})
            return cand
    return None                                               # 顺延窗口内仍无空位 → 上层 raise_conflict('NO_MACHINE')
```

- **要点**:① 顺延**先同日后续班次,再后续透析日**,优先保持病人原班次以减少作息扰动;② 命中只生成**建议草稿 + HINT 队列**,最终由护士长人工确认,系统不自动定;③ 顺延窗口受 `horizon_days` 约束,避免无限外推;窗口内全满才升级为 `NO_MACHINE` 报警。

## B.2 周序号 / 奇偶周(决策 21,规范 §1.3/§5)+ HDF 替换判定 + 入组定 parity(决策 1/3/21)

> **决策 21(D-6)锚点定义**:全局固定基准周一 `OddEvenWeekAnchorMonday`(配置项,默认 2025-01-06)。
> 周序号 = 连续整数计数,**跨年无缝、永不漂移**;奇偶 = 序号 mod 2。
> 奇偶周**仅服务 HDF 错峰**;5 种频率模式本身不看单双周。

```python
def monday_of(d):
    """该日期所在周的周一(ISO:周一=1 … 周日=7)。"""
    return d - days(weekday_iso(d) - 1)               # weekday_iso: Mon=1..Sun=7

def week_index(tenant, d):
    """全局连续周序号:以固定基准周一为第 0 周,逐周 +1,可跨年、可为负(基准前)。"""
    anchor = config(tenant, 'OddEvenWeekAnchorMonday')   # 默认 2025-01-06(周一)
    return (monday_of(d) - anchor).days // 7             # 整除:同一周内任意日得同一序号

def week_parity(tenant, d):
    """奇偶:0=偶数周 / 1=奇数周。Python 取模对负数仍非负,基准前的周也稳定。"""
    return week_index(tenant, d) % 2                      # 0 或 1,与 HdfWeekParity 取值一致
```

- **正确性要点**:① 锚点必须是周一,`week_index` 才在同一自然周内对周一~周日给出同一整数;② 用 `monday_of` 把任意日归一到本周周一再相减,避免"日内偏移"影响整除;③ 取模口径与 `HdfWeekParity`(0/1)一致,`decide_mode_of_this_session` 直接 `==` 比较即可;④ 整数序号连续递增,跨年/跨多次生成不会重置,从根本上消除"漂移"。

```python
def decide_mode_of_this_session(tenant, item, due_on):
    """HDF 是替换:仅当 (本日==HDF日) 且 (本周parity==该病人HDF所属周parity) 时,本次=HDF,否则=HD。"""
    if not item.hdf_enabled:
        return 'HD'
    if weekday(due_on) != item.hdf_weekday:
        return 'HD'
    if week_parity(tenant, due_on) != item.hdf_week_parity:   # 奇/偶周错峰,口径同 week_parity
        return 'HD'
    return 'HDF'                                              # 这一次的 HD 被替换为 HDF,总次数不变

def assign_hdf_week_parity(tenant, template, dates):
    """决策21:HDF 病人入组时分到当前 HDF 机负载较轻的那一侧(奇/偶),之后每两周做一次 HDF。
       已排定的保持不变(尽量固定);新入组者按当前负载选较轻 parity。"""
    # 统计每个 (HDF日, parity) 上,固定 HDF 机的现有占用数;parity 取值 0/1,与 week_parity 对齐
    load = defaultdict(int)   # key=(weekday, parity) -> count
    for it in template.hdf_items():
        if it.hdf_week_parity is not None:                   # 已排定:计入负载、不再变动
            load[(it.hdf_weekday, it.hdf_week_parity)] += 1
    for it in template.hdf_items():
        if it.hdf_week_parity is not None:
            continue                                         # 跳过已排定者(决策3"一旦排定尽量固定")
        even = load[(it.hdf_weekday, 0)]                     # 0=偶数周一侧负载
        odd  = load[(it.hdf_weekday, 1)]                     # 1=奇数周一侧负载
        chosen = 0 if even <= odd else 1                     # 选当前较轻的一侧(均匀错峰)
        it.hdf_week_parity = chosen
        persist_parity(it.patient, chosen)                   # 写回 Profile.HdfWeekParity / TemplateItem
        load[(it.hdf_weekday, chosen)] += 1
    # 注:某 (HDF日,parity) 两侧 HDF 机都不够才微调(改 weekday),再不行 → raise_conflict('HDF_NO_MACHINE')
```

## B.4 挑机偏好:集中连片 + 组团(决策 10,规范 §4.4)与三级确认

```python
def pick_best(candidates, patient, cell):
    """多台候选时打分挑:集中(保护整台全空机)+ 组团(同班同方案相邻)。不做负载均衡。"""
    def score(m):
        s = 0
        # 组团:相邻机位若已坐同班同方案病人 → 加分(便于护士分组)
        for nb in neighbors(m, cell.ward):                       # PositionIndex ±1
            occ = occupant(nb, cell)
            if occ and occ.shift == patient.shift and occ.mode == cell_mode(patient, cell):
                s += 10
        # 集中连片:优先填"已部分占用的连片块",保护"整台全空机"留给新病人三天同机
        s += contiguity_bonus(m, cell)
        # 固定位记忆:靠近病人历史机位
        s += proximity_bonus(m, patient)
        return s
    return max(candidates, key=score)
```

```python
        return   # 拒收:根本不生成排班(规范 §7.2)
    days = due_dates_of(patient, weeks)        # 该病人这两周的所有透析日(按 FreqPattern)
    # 1) 首选:找一台所有透析日都空闲、且机型匹配、同区同班的机器,一次性固定
    for m in machines(patient.home_ward, type_for(patient), order=by_contiguity):
        if all(is_free(m, Cell(patient.home_ward, patient.shift, d)) for d in days):
            fix_machine(patient, m)            # 写 FixedHdMachine(若 HDF 另定 FixedHdfMachine)
            for d in days: draft(patient, Cell(..., d), m, source='REGULAR')
            return
    # 2) 退路:逐次分配(各日分别找空位,可能不同机)
    placed_any = False
    for d in days:
        cell = Cell(patient.home_ward, patient.shift, d)
        m = place_hd_session(tenant, cell, SessionItem(patient, mode_on(patient,d)))
        if m: draft(patient, cell, m, source='REGULAR'); placed_any = True
        else: raise_conflict('NEW_PATIENT_UNPLACED', patient, cell)
    # 3) 完全排不开 → 已在逐次分支内 raise_conflict 报警人工
```

# C. 关键设计决策与权衡

1. **机器/位分离,不再用"床表"**:规范明确"机器有机型+停用时段""一班一机=一位"。把老系统的 `Schedule_Bed` 升级为 `Schedule_Machine`(带 `MachineType`/`PositionIndex`),位由"机器×班次×日期"动态计算。好处:机型能力、双固定机位、连片打分都有自然落点;代价:与老 `BedId` 不直接对齐,迁移需影子列。

2. **病人骨架独立成 `Schedule_PatientProfile`(1:1)**:规范反复强调"骨架是人工权限属性,算法只读"。独立表把排班域属性(标签/频率/班次/双固定机/HDF日)与主病人档分离,权限边界清晰,算法只读不写这些字段(只写 `FixedHd/HdfMachineId` 作为"记忆",且仅在自动固定或人工特批时)。

3. **模板独立两表(头+项),彻底弃用 Status=60**:这是规范的硬要求(根除"删排班误删模板")。模板只存稳定病人骨架,生成=往后铺;变动病人单独处理,不全量重排——保护位置稳定性、省算力。

4. **三级确认 = 一个"已确认"状态 + 三个时间戳**(决策16),而非三个状态:状态机保持七态简洁,确认进度由 `Confirm1/2/3At` 表达,且记录确认人(权限留痕)。二次确认的"前一天"非工作日跳转在算法层 `prev_dialysis_day` 处理,不污染数据模型。

5. **冲突队列是"主→备→报警"的统一落点**:所有自动排不开的分支都写 `Schedule_ConflictQueue`,绝不硬排。`ConflictType` 枚举覆盖 HDF 无机/区满/新病人/停机/假日/方案变更。系统给 `SuggestedShiftId` 建议草稿,人工确认后转正——完全贴合"系统建议+人工确认,不自动定"。

6. **缺席(80)与取消(70)分离 + `IsBorrowedSlot`**:满足"缺席留痕、机位可借、不改原归属";`MakeupOfShiftId` 让人工补排可追溯,但补不补由人工决定(系统不自动补)。

7. **CRRT 主记录走状态机 + 扩展表存时间区间**:CRRT 不走三班/模板,但仍需在统一视图里显示与做机器占用冲突检测,故保留一条 `PatientShift`(RecordForm=20)主记录 + `Schedule_CrrtSession`(机+起止时间)。

8. **频率用五种枚举 `FreqPattern` 替代老系统奇偶周次数**:规范 §2.1 把频率固化为五种命名模式,直接推导透析日,语义比 `OddWeek/EvenWeekFrequency` 更强;迁移时做映射,映射不上的入队列人工确认。

9. **奇偶周用"全局固定基准周一 + 连续计数"(决策 21)**:不用 ISO 周、不用"生成起始日所在周",而是配置一个永久基准周一(默认 2025-01-06),`week_index = (本周周一 − 基准)/7`、`parity = index % 2`。连续整数计数使奇偶**跨年无缝、跨多次生成不漂移**;`HdfWeekParity` 在病人入组时按当前 HDF 机两侧负载选较轻一侧,达成机位均匀错峰。该字段仅用于 HDF,不影响 5 种频率模式的透析日推导。

10. **取消"加床",排满走时间维度顺延(决策 22)**:容量模型严格守"机器数 × 班次数",**不引入任何临时位/机动机/`ExtraSlot` 载体**。某格排满时,先在后续班次、再在后续日期(优先病人原班次)找空位,生成建议草稿交护士长人工确认;顺延窗口(默认 14 天)内仍无空位才报 `NO_MACHINE`。好处:模型简单、不破坏"一班一机=一位"、所有溢出都有人工兜底;代价:极端拥挤时病人可能被顺延到非首选时段(以 HINT 队列提示)。
