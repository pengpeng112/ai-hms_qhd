## 二、数据库结构

### 2.1 表清单 (14 张核心表)

所有表直接复用老库 `Schedule_*` 系列表，不再使用 `Schedule_v2_*` 前缀。

| 表名 | GORM 模型 | 说明 |
|------|-----------|------|
| `Schedule_Ward` | `model.Ward` | 病区 (A-1/A-2/B/C)，新增 ZoneType/IsSubZone 列 |
| `Schedule_Bed` | `model.Machine` | 机器 (老库用 Bed 表表达机器)，新增 MachineType/SupportedModes/Code/PositionIndex |
| `Schedule_MachineOutage` | `model.MachineOutage` | 机器停机记录 |
| `Schedule_Shift` | `model.Shift` | 班次 (早/中/夜)，新增 ShiftCode |
| `Schedule_Calendar` | `model.Calendar` | 机构日历 (透析日/假日) |
| `Schedule_Patient` | `model.Patient` | **新表**—病人主档 (快照自 Register_PatientInfomation)，含院感状态 |
| `Schedule_PatientProfile` | `model.PatientProfile` | 病人排班骨架 (频率/班次/模式/HDF/固定机位)，新增 WeeklyCount/PatientStatus 等 |
| `Schedule_PlanChange` | `model.PlanChange` | 方案变更生效记录 |
| `Schedule_PatientShift` | `model.PatientShift` | **核心表**—排班记录，老库列名 TreatmentTime (非 ScheduleDate) |
| `Schedule_CrrtSession` | `model.CrrtSession` | CRRT 占用记录 |
| `Schedule_ScheduleTemplate` | `model.ScheduleTemplate` | 排班模板头 |
| `Schedule_ScheduleTemplateItem` | `model.ScheduleTemplateItem` | 模板项，新增 DefaultMode/MachineId |
| `Schedule_ConflictQueue` | `model.ConflictQueue` | 冲突/待处理队列 |
| `Schedule_TenantSetting` | `model.TenantSetting` | 租户级配置 (奇偶周锚点/顺延窗口等) |

### 2.2 关键列映射 (老库兼容)

| 业务概念 | 老库列名 | 说明 |
|----------|----------|------|
| 排班日期 | `TreatmentTime` (timestamp) | 不是 `ScheduleDate`！所有 SQL 都用双引号 `"TreatmentTime"` |
| 机器 | `Schedule_Bed` 表 | 不是 `Schedule_Machine`！ |
| 机器ID | `MachineId` (int64, NOT NULL DEFAULT 0) | 与 `BedId` 同值 |
| 班次ID | `ShiftId` (int64, NOT NULL DEFAULT 0) | CRRT 用 0 表示无班次 |
| NULL 判空 | `"ShiftId" > 0` 等 | 不能用 `IS NOT NULL` (因为是 int64 NOT NULL) |

### 2.3 唯一索引 (repo.EnsureIndexes 创建)

```sql
-- 同病人同日同班次唯一
CREATE UNIQUE INDEX IF NOT EXISTS uq_ps_patient_slot
  ON "Schedule_PatientShift" ("TenantId", "PatientId", "TreatmentTime", "ShiftId")
  WHERE "ShiftId" > 0 AND "Status" NOT IN (70, 80);

-- 同机同日同班次唯一
CREATE UNIQUE INDEX IF NOT EXISTS uq_ps_machine_slot
  ON "Schedule_PatientShift" ("TenantId", "MachineId", "TreatmentTime", "ShiftId")
  WHERE "MachineId" > 0 AND "ShiftId" > 0 AND "Status" NOT IN (70, 80);
```

#### 治疗执行 ★新增

| 方法 | 路径 | 角色 | 说明 |
|------|------|------|------|
| POST | `/shifts/:id/start` | 护士/主班/护士长 | **上机**: Status 20→50，含院感闸门 + 当日校验 |
| POST | `/shifts/:id/complete` | 护士/主班/护士长 | **下机**: Status 50→60，写 Confirm3At |

**上机闸门逻辑 (infectionGate)**：
```
InfectionStatus = "negative" → 放行
InfectionStatus = "positive" + Profile.ZoneTag = "C" → 放行, 否则 → 409 "确诊传染病,仅C区可收治"
InfectionStatus = "unknown" + InfectionWaivedAt != nil → 放行, 否则 → 409 "院感指标未出,需护士长确认豁免"
```

**上机错误码**:
- `ErrNotConfirmed` (409): 排班未确认
- `ErrNotToday` (409): 非透析当日
- `ErrInfectionUnconfirmed` (409): 院感指标未出
- `ErrInfectionPositive` (409): 阳性非C区

**下机错误码**:
- `ErrNotInDialysis` (409): 不在透析中状态

### 3.3 主系统治疗流程同步钩子 ★新增

主系统 `TreatmentService` 在治疗状态变更时自动同步 `Schedule_PatientShift.Status`：

| 触发点 | 同步目标 | 说明 |
|--------|----------|------|
| `UpdateStatus(id, InProgress)` 成功后 | `Schedule_PatientShift.Status = 50` | 上机同步，通过 `Treatment_Treatment.ScheduleId` 查找 |
| `SubmitPostAssessment(id)` 成功后 | `Schedule_PatientShift.Status = 60` | 下机同步 |

同步失败**不阻断**主流程（仅 `log.Printf` 警告）。

## 四、排班状态机

```
0(待排) ←── 缺口，应排未排
    ↓ generate
10(草稿) ←── 模板复制/算法生成
    ↓ confirm-plan
20(已确认) ←── 计划生效 (Confirm1/2/3At 表达确认进度)
    ↓ start ★新增           ↓ cancel
50(透析中) ──绿色光晕──    70(已取消) ──留痕──
    ↓ complete ★新增        ↓ absent
60(已完成) ──半透明──      80(缺席) ──留痕──
```

**编辑保护 (`canEdit`)**：
- Status=50/60 → 锁定 (返回 409)
- ScheduleDate < 今天 → 锁定

## 六、硬规则与约束

1. **禁止 AutoMigrate**: `internal/database/migrate.go` 全局禁止，所有 DDL 需手动 SQL 脚本
2. **GORM 配置**: `SingularTable: true`, `NoLowerCase: true` — 表名/列名大小写敏感，SQL 必须用双引号
3. **老库列名**: `TreatmentTime` (不是 `ScheduleDate`), `Schedule_Bed` (不是 `Schedule_Machine`)
4. **NULL 处理**: `int64 NOT NULL DEFAULT 0` 替代 `*int64`，判空用 `> 0` 不用 `IS NOT NULL`
5. **路由挂载**: v2 API 挂载在 `/api/v2` 下，通过 `gin.RouterGroup` 注入
6. **响应包装**: `responseWrapper()` 自动将裸 JSON 包装为 `{success: true, data: ..., timestamp: ...}`
7. **鉴权**: JWT → `tenantMiddleware()` → `tenant`/`role`/`userId` 注入 Context
8. **租户**: 硬编码 `LegacyTenantID = 3`
9. **开发模式**: `DEV_SUPERUSER=true` 跳过角色校验 (生产禁止)
10. **CORS**: 需要 `CORS_ALLOWED_ORIGINS` 环境变量
