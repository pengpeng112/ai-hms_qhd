# AI-HMS 透析执行界面原布局轻量优化方案

## 一、设计原则

本方案不再采用“大改三栏工作台/图标侧栏”的方式，而是保持当前页面原有结构：

- 左侧主菜单保持原宽度和分组方式
- 患者列表仍在左侧第二栏
- 顶部步骤条位置不变
- 患者摘要栏位置不变
- “暂无治疗记录”提示位置不变
- 体重与容量、生命体征、血管通路等表单仍纵向排列
- 底部保存/提交操作位置保持原有习惯

只做以下轻量调整：

1. 优化菜单层级和行高
2. 压缩患者列表高度和间距
3. 压缩顶部步骤按钮高度
4. 压缩患者摘要栏高度
5. 缩小“暂无治疗记录”提示卡高度
6. 表单卡片从大间距改为紧凑卡片
7. 表单字段统一高度、统一单位位置
8. 增强必填项、自动获取项、风险提示的视觉区分
9. 底部操作栏强化但不改变原功能
10. 减少滚动，让核心内容尽量一屏可见

---

## 二、当前布局保留范围

### 保留

- 左侧深色菜单
- 患者队列
- 顶部步骤按钮
- 患者基本信息区
- 治疗记录提示区
- 体重与容量评估
- 生命体征监测
- 血管通路与神志状态
- 底部接诊医生、评估人、提交按钮
- 现有保存、暂存、提交、打印、返回逻辑

### 不做

- 不改成 84px 图标菜单
- 不改成右侧固定摘要栏
- 不重排为全新的三栏工作台
- 不修改接口
- 不修改字段名
- 不删除原有按钮
- 不影响患者切换、步骤切换、创建治疗记录和保存

---

## 三、具体优化内容

### 1. 左侧菜单轻量优化

保持当前宽度和分组，只调整视觉：

- 分组标题高度控制在 30px
- 菜单项高度控制在 36px
- 当前选中项保留蓝色背景
- 左侧增加 3px 高亮条
- 字体减小一级，减少纵向占用
- 如果存在重复分组，建议后续再清理，但本次不强制改逻辑

样式建议：

```css
.sidebar-menu-group {
  height: 30px;
  padding: 0 12px;
  font-size: 12px;
  color: #8ea4c5;
}

.sidebar-menu-item {
  height: 36px;
  border-radius: 10px;
  font-size: 13px;
}

.sidebar-menu-item.active {
  background: #1d63ff;
  color: #fff;
  box-shadow: inset 3px 0 0 #80b7ff;
}
```

---

### 2. 患者列表轻量优化

保持患者列表位置，只减少高度和噪声：

- 筛选条高度 34px
- 搜索框高度 38px
- 患者行高度 46px
- 当前患者浅蓝底 + 左侧蓝条
- 状态 tag 控制在 22px 高

患者项展示：

```text
支英俊
女 / 72岁                    未排床
```

---

### 3. 顶部步骤条优化

保持顶部步骤横向按钮，不改变功能：

- 按钮高度从约 40px 降到 36px
- 字体 12px
- 当前步骤蓝色实心
- 其他步骤浅灰背景
- 间距 10px

---

### 4. 患者摘要栏优化

保持原位置，压缩为 96px 左右：

左侧：

```text
支英俊  未排床
患者ID 300410
性别/年龄 女/72岁
费用类别 市职工普通
透龄 待补充
```

右侧：

```text
干体重 0kg
治疗方案 HD
打印
返回
```

优化点：

- 白色卡片
- 浅灰边框
- 柔和阴影
- 患者姓名更突出
- 指标用小卡片承载

---

### 5. 暂无治疗记录提示优化

保持提示位置，但高度缩小到 72px 左右：

```text
暂无治疗记录
可先创建今日治疗记录，再继续录入和查看。
[创建治疗记录]
```

不要占用过多首屏空间。

---

### 6. 体重与容量评估卡片优化

保持原卡片顺序，但高度压缩到约 198px：

字段一行展示：

```text
透前体重 | 额外体重 | 干体重 | 体重增长 | 目标超滤量
```

第二行：

```text
患者拒测 / 卧床
称重照片历史：暂无称重照片（功能完善后可补录）
```

优化点：

- 字段高度 38px
- 单位放输入框右侧
- 必填字段前保留 *
- 自动获取字段用浅灰 placeholder
- 称重照片区域不再大块占位

---

### 7. 生命体征监测卡片优化

高度控制在 170px 左右。

字段第一行：

```text
透前血压 | 测压部位 | 透前心率 | 透前体温 | 呼吸
```

第二行：

```text
疼痛评分 | 风险提示
```

风险提示：

```text
提示：血压、心率、体温异常时，保存前请复核。
```

---

### 8. 血管通路与神志状态卡片优化

高度控制在 180px 左右。

字段第一行：

```text
A端位点 | V端位点 | 神志状态 | 护理分级
```

第二行：

```text
血管通路既往史 | 透前备注
```

---

### 9. 底部操作区优化

保持原底部操作区，但强化可见性：

```text
接诊医生：test_admin
评估人：test_admin
当前患者：支英俊（未排床）
必填未完成：4 项

恢复草稿 | 暂存草稿 | 提交透前评估
```

建议：

- 白底卡片
- 固定在当前内容底部
- 主按钮蓝色
- 次按钮白色/浅灰

## 六、验收清单

1. 左侧菜单结构未大改。
2. 患者列表位置未变。
3. 顶部步骤按钮功能未变。
4. 患者摘要信息仍完整。
5. 创建治疗记录按钮正常。
6. 体重与容量字段可录入。
7. 生命体征字段可录入。
8. 血管通路字段可录入。
9. 保存、暂存、提交按钮功能不受影响。
10. 打印、返回功能不受影响。
11. 页面滚动明显减少。
12. 1366x768 下可见内容更多。
13. 控制台无 JS 报错。
