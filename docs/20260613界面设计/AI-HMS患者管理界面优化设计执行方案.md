# AI-HMS 患者管理界面优化设计执行方案

适用页面：`ai-hms-frontend/src/pages/PatientList.tsx`  
目标：不修改接口、不修改字段、不改变搜索、筛选、分页、新增建档、删除和详情跳转逻辑，只优化“全科患者管理”界面的样式、展现层级和日常使用效率。

---

## 1. 当前代码结构核查

当前 `PatientList.tsx` 已经具备完整患者列表能力：

1. 页面状态包括：
   - `searchTerm`
   - `debouncedSearch`
   - `activeFilter`
   - `loading`
   - `patients`
   - `apiError`
   - `currentPage`
   - `total`
   - `createModalOpen`
   - `refreshKey`
   - `patientStats`

2. 当前筛选类型：
   - `all`
   - `today`
   - `active`
   - `mine`
   - `in_dept`
   - `transferred`

3. 当前接口能力：
   - `restApi.getPatientList(params)`
   - `restApi.getPatientStats()`
   - `restApi.deletePatient(patientId)`

4. 当前已有功能：
   - 搜索防抖
   - 后端分页
   - 全部 / 在科 / 转出等筛选
   - 更多筛选：今日治疗、透析中、我的患者
   - 新增建档弹窗
   - 删除患者
   - 复制 ID
   - 点击行进入患者详情

因此本次只做前端 UI 和交互展示优化，不动 API。

---

## 2. 当前界面主要问题

从截图看，当前患者管理界面整体清爽，但还存在以下问题：

### 2.1 统计卡信息有，但层级偏弱

当前统计卡显示：

```text
总人数
在科活跃
门诊
住院
```

但卡片比较平，缺少说明和颜色层级。用户一眼能看到数字，但不容易快速判断哪个是重点。

---

### 2.2 搜索、刷新、新增位置较分散

当前搜索框和新增按钮在右上角，能用，但搜索区域和页面标题之间缺少“患者管理工具栏”的感觉。

建议把页面头部变成：

```text
标题 + 说明 + 刷新 + 搜索 + 新增建档
```

统一到一个白色页头里。

---

### 2.3 筛选 Tab 不够完整直观

当前主卡中显示：

```text
全部患者
在科患者
转出患者
```

其他筛选藏在右侧漏斗菜单里：

```text
今日治疗
透析中
我的患者
```

建议把高频筛选全部显性展示：

```text
全部患者
在科患者
今日治疗
透析中
我的患者
转出患者
```

漏斗按钮保留，用于后续高级筛选。

---

### 2.4 表格行信息略松散

当前表格行高度较高，信息可读，但一屏可见患者数量不够多。患者管理通常需要快速浏览，建议压缩行高并增强信息分区。

---

### 2.5 主治医生为空时不够醒目

截图中主治医生列基本为空。建议空值统一显示：

```text
--
```

并使用灰色，避免界面看起来像数据错位。

---

### 2.6 操作列不够直观

当前操作是：

```text
更多 ...
箭头 →
```

功能上没问题，但用户不一定知道 `...` 里有什么。建议把常用操作显性化：

```text
复制ID
删除
进入详情 →
```

如果担心过多，可以桌面端显性，窄屏仍使用更多菜单。

## 4. 具体优化建议

### 4.1 顶部页头优化

当前标题区建议改成白色卡片：

```text
全科患者管理
统一维护患者档案、透析方案、医保类型、当前状态与床位信息。

[刷新] [搜索姓名、ID、床号...] [+ 新增建档]
```

优点：

1. 页面入口更正式。
2. 搜索和新增集中。
3. 刷新按钮不再孤立。

---

### 4.2 统计卡优化

保留原统计字段，但增加说明和顶部色条：

```text
总人数 365
全部建档患者

在科活跃 102
当前在院/在科患者

门诊 0
门诊透析患者

住院 0
住院患者
```

样式建议：

```tsx
<div className="rounded-2xl border border-slate-200 bg-white p-5 shadow-sm">
  <div className="h-1 rounded-t-2xl bg-blue-600" />
  <div className="text-3xl font-black">{value}</div>
  <div className="text-sm font-bold text-slate-500">{label}</div>
  <div className="text-xs text-slate-400">{desc}</div>
</div>
```

---

### 4.3 筛选区优化

将原来藏在 Dropdown 里的高频筛选展示出来。

建议：

```tsx
const filterItems = [
  { key: 'all', label: '全部患者', count: patientStats?.totalCount },
  { key: 'in_dept', label: '在科患者', count: patientStats?.activeCount },
  { key: 'today', label: '今日治疗', count: undefined },
  { key: 'active', label: '透析中', count: undefined },
  { key: 'mine', label: '我的患者', count: undefined },
  { key: 'transferred', label: '转出患者', count: undefined },
]
```

展示为胶囊按钮：

```text
全部患者 365
在科患者 102
今日治疗
透析中
我的患者
转出患者
```

保留高级筛选按钮：

```text
筛选条件 ⌄
```

---

### 4.4 表格行优化

当前表格列不建议大改，保留：

```text
基本信息
类型 / 医保
治疗方案
当前状态
主治医生
快捷操作
```

但做以下优化：

1. 行高控制在 68–76px。
2. 患者姓名加粗。
3. 患者 ID、性别、年龄在一行展示。
4. 类型、医保上下排列。
5. 治疗方案展示为：
   ```text
   HD | 48kg
   ```
6. 状态显示 badge，床号显示小标签。
7. 主治医生空值显示灰色 `--`。
8. 行 hover 时左侧出现蓝色高亮条。

---

### 4.5 操作列优化

当前操作保留更多菜单，但建议桌面端显性展示：

```text
复制ID   删除   进入详情 →
```

其中：

1. 复制 ID 保留 `navigator.clipboard.writeText`。
2. 删除仍然走 `Modal.confirm`。
3. 点击整行仍然进入详情。
4. 操作按钮点击继续 `stopPropagation()`，避免触发行点击。

窄屏时仍可使用 `Dropdown`。

---

### 4.6 空状态优化

当前空状态只有：

```text
未找到符合条件的患者
```

建议增强为：

```text
未找到符合条件的患者
请调整搜索关键词或清空筛选条件
[清空筛选]
```

第一版可以只改文案，不一定新增按钮。

---

### 4.7 分页条优化

当前分页可用，但建议增加页大小提示：

```text
显示 50 条，共 102 条    50 条/页    上一页  第 1 页  下一页
```

保持现有 `pageSize = 50` 不变。

---

## 6. 组件化建议

当前 `PatientList.tsx` 已经比较长。为了降低风险，第一版可以先在单文件内改样式。

如果后续想继续优化，可拆为：

```text
PatientStatsCards.tsx
PatientFilterBar.tsx
PatientTable.tsx
PatientRow.tsx
PatientPagination.tsx
```

第一版不建议强制拆分，避免影响业务。
