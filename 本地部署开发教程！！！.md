很好，你的 PostgreSQL 已经有了（`admin/admin123`），那我们按“**一定能跑起来**”的方式来，一步一步做。  
下面是 Windows + PowerShell 的实操流程（含排查方法）。

---

## 一、先做环境自检（避免后面卡住）

### 1）检查 PostgreSQL 是否在跑
在 PowerShell 执行：

```powershell
Get-Service *postgres*
Test-NetConnection -ComputerName 127.0.0.1 -Port 5432
```

- `TcpTestSucceeded = True` 才说明 5432 可连。

### 2）检查工具版本

```powershell
go version
node -v
npm -v
psql --version
```

建议：
- Go \(>=1.24\)
- Node \(>=20\)
- npm \(>=10\)

> 如果 `psql` 不识别，说明没加 PATH。可以用 pgAdmin 执行 SQL，或者把 PostgreSQL `bin` 目录加到 PATH。

---

## 二、创建业务数据库（ai_hms_db）

你现在是 `admin/admin123`，先用这个连默认库 `postgres`：

```powershell
psql -h 127.0.0.1 -p 5432 -U admin -d postgres
```

输入密码：`admin123`

进入后执行：

```sql
SELECT current_user;
CREATE DATABASE ai_hms_db;
\c ai_hms_db
GRANT ALL PRIVILEGES ON DATABASE ai_hms_db TO admin;
GRANT ALL ON SCHEMA public TO admin;
```

退出：

```sql
\q
```

### 如果这里报错怎么排查
- `FATAL: password authentication failed`  
  → 用户名/密码不对，先在 pgAdmin 里确认账号。
- `permission denied to create database`  
  → 当前账号无建库权限，让 DBA 执行：  
  `ALTER USER admin CREATEDB;`
- `database "ai_hms_db" already exists`  
  → 可忽略，继续下一步。

---

## 三、配置并启动后端（自动建表关键步骤）

项目后端目录：`f:\python\前后端代码\ai-hms_qhd\ai-hms-backend`

### 1）生成 `.env`

```powershell
Set-Location "f:\python\前后端代码\ai-hms_qhd\ai-hms-backend"
Copy-Item ".env.example" ".env" -Force
```

### 2）编辑 `.env`（最重要）

确保至少这些值：

```env
SERVER_PORT=8080
SERVER_HOST=0.0.0.0
GIN_MODE=debug

DB_HOST=127.0.0.1
DB_PORT=5432
DB_USER=admin
DB_PASSWORD=admin123
DB_NAME=ai_hms_db
DB_SSL_MODE=disable

JWT_SECRET=your_jwt_secret_key_here
APP_SECRET=your_app_secret_for_encryption

CORS_ALLOWED_ORIGINS=http://localhost:5173,http://localhost:3000
```

> 必须是 `GIN_MODE=debug`，因为这个项目在 debug 才执行 AutoMigrate 自动建表。

### 3）拉依赖并启动

```powershell
go mod download
go run cmd/server/main.go
```

### 4）判断是否启动成功
另开一个 PowerShell：

```powershell
curl http://localhost:8080/health
```

返回成功 JSON 即后端 OK。

---

## 四、确认表结构是否已经自动创建

后端跑起来后，执行：

```powershell
psql -h 127.0.0.1 -p 5432 -U admin -d ai_hms_db -c "SELECT tablename FROM pg_tables WHERE schemaname='public' ORDER BY tablename;"
```

你应看到很多表（如 `patients`, `users`, `treatments`, `dict_items` 等）。

也可以查总数：

```powershell
psql -h 127.0.0.1 -p 5432 -U admin -d ai_hms_db -c "SELECT count(*) FROM pg_tables WHERE schemaname='public';"
```

### 如果表没创建，重点排查
1. `.env` 是否生效（特别是 `GIN_MODE=debug`）
2. 后端控制台是否有：
   - `failed to connect to database`
   - `Failed to migrate database`
3. 数据库账号是否有 schema 权限（执行过 `GRANT ALL ON SCHEMA public TO admin`）

## 五、导入测试数据（可选但推荐）

后端目录执行：

```powershell
psql -h 127.0.0.1 -p 5432 -U admin -d ai_hms_db -f "scripts/test_data.sql"
```

验证有无测试患者：

```powershell
psql -h 127.0.0.1 -p 5432 -U admin -d ai_hms_db -c "SELECT id,name,status FROM patients WHERE id LIKE 'test-%';"
```

> 不建议直接用 `import_test_data.sh`（它脚本里写死了用户名）。

## 六、配置并启动前端

目录：`f:\python\前后端代码\ai-hms_qhd\ai-hms-frontend`

### 1）确认开发环境 API 地址
编辑 `.env.development`，确保是：

```env
VITE_API_BASE_URL=http://localhost:8080
```

> 这个项目前端代码读取的是 `VITE_API_BASE_URL`（不是 `VITE_BACKEND_API_URL`）。

### 2）安装并启动

```powershell
Set-Location "f:\python\前后端代码\ai-hms_qhd\ai-hms-frontend"
npm ci
npm run dev
```

访问：`http://localhost:5173`

登录测试账号：`admin / admin123`

## 七、前后端连通与数据连通验证（最小闭环）

1. 浏览器打开登录页成功  
2. 登录成功跳转主页  
3. 患者列表能看到数据（或至少请求成功）  
4. F12 Network 里接口请求是到 `http://localhost:8080/...`  
5. 后端终端有对应请求日志  
6. PostgreSQL 中数据查询有变化（如新增患者后 `patients` 记录增加）

## 八、常见问题速查表（遇到问题按这个定位）

### A. 后端启动了但提示数据库连接失败
- 看 `.env` 的 `DB_HOST/PORT/USER/PASSWORD/DB_NAME`
- `Test-NetConnection 127.0.0.1 -Port 5432`
- 用同样参数跑 `psql` 测试登录

### B. 后端能启动但没表
- 是否 `GIN_MODE=debug`
- 启动日志是否有 migration error
- 账号是否有 public schema 权限

### C. 前端打开了但接口 401/403
- 当前登录是后端硬编码账号 `admin/admin123`
- 看浏览器 localStorage 是否有 `hdis_access_token`
- 看请求头是否带 `Authorization: Bearer ...`

### D. 前端请求报 CORS
- 后端 `.env` 的 `CORS_ALLOWED_ORIGINS` 是否包含 `http://localhost:5173`
- 改完后端要重启

### E. 前端请求地址不对（跑到线上地址）
- 检查 `.env.development` 的 `VITE_API_BASE_URL`
- 重启 `npm run dev`（Vite 环境变量变更需要重启）
